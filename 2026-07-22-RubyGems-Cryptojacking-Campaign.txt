
2026-07-22 (THURSDAY): RubyGems Cryptojacking Campaign — Trojanized Popular Gems with XMRig Miner

AUTHOR(S):

NOTES:

ACTOR 1 — Prvaz12_mars (jdvrie98@gmail[.]com):

- 113 malicious gems were published on July 13, 2026, reaching 14,253 total downloads.
- Each gem is a trojanized copy of a popular, legitimate Ruby library. 
     - The naming pattern: {giga,nano,ultra,hyper,piko,super,mega,micro,mini,tiny}-{pure,sharp,smart,pro,safe,clean,fast,quick,lite,max}-{kit,tool,mod,pkg,hub,lib,box,rb,sys,gem}.
- All gems share an identical XMRig cryptojacking payload injected into the bundled library's entry point file.
- The payload uses a 5-hour delayed Thread.new{sleep 18000; ...} trigger to evade sandbox analysis.
- Persistence through three mechanisms: appending to ~/.bashrc, installing a crontab entry, and creating a systemd user service.
- SSH lateral movement: reads ~/.ssh/id_rsa and ~/.ssh/known_hosts, then propagates to known hosts.
- Cross-language worm: infects local Node.js (package.json postinstall), Python (setup.py), Ruby (.gemspec), Docker (Dockerfile), git (pre-commit hooks), and VSCode extensions.
- Anti-analysis: detects VMs (VMware, VirtualBox, QEMU), containers (/.dockerenv, cgroup), and debuggers before executing.
- Load-aware mining: pauses when system load is high; CPU max-threads-hint capped at 50%.
- Encrypted config: AES-256-GCM + RSA-4096 + ChaCha20 triple-layer encryption.
- All gems mine to the same Monero wallet and 4 mining pools (moneroocean, p2pool, supportxmr, herominers).

ACTOR 2 — monib110 (nsrtytyam687@gmail[.]com):

- 23 malicious gems were published as of 2026-07-16.
- Initially published clean placeholder gems on Jul 12 (same naming pattern, description "University research based on...").
- Weaponized all 23 packages on July 16, introducing a simplified XMRig payload.
- Payload is a hidden file lib/.threadpool.rb loaded via require_relative '.threadpool' from the main entry point.
- No obfuscation — plaintext Ruby code with XMRigLauncher module.
- Downloads XMRig dynamically via GitHub API (api.github.com/repos/xmrig/xmrig/releases/latest).
- Runs miner inline via exec() — no persistence, no worm, no anti-analysis.
- Single pool: pool.supportxmr.com:3333
- Uses two naming strategies: (1) same adj-noun pattern as Prvaz12_mars, (2) typosquats with character substitution (i→1, o→0, doubled letters).
Typosquat targets: minitest, rake, activesupport, addressable, json, aws-partitions, aws-sdk-core, aws-sigv4, jmespath, i18n, aws-eventstream, rack, tzinfo, faraday, public_suffix, rspec-core, http-cookie, multipart-post, bundler.
